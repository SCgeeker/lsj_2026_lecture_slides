Part 1：認識 jamovi 介面

jamovi 主畫面

  • 左上：檔案選單（三條橫線 ☰）
  • 左側：資料表 (Data)
  • 右側：分析結果 (Results)
  • 上方：分析選單 (Analyses)

五大工作面板(Tab)

面板 功能
Variable 檢視與編輯資料變項
Data 檢視與編輯資料
Analyses 執行統計分析
Plots 製作資料視覺化(v 2.7)
Results/Edit 顯示與編輯分析結果

Part 3：設定變項類型

為什麼要設定變項？

jamovi 會「猜測」變項類型

但常常猜錯！

  • 你需要告訴 jamovi 每個變項的正確類型

開啟變項編輯器

操作方式：

  1. 雙擊變項標題列
  2. 或點選 Data → Setup

測量尺度 (Measure Type)

類型 圖示 說明 範例
ID 識別碼 學號、編號
Nominal 名義（類別） 性別、組別
Ordinal 次序 滿意度等級
Continuous 連續數值 分數、身高

資料型態 (Data Type)

型態 說明
Text 文字
Integer 整數
Decimal 小數

實作練習（Part 3）

請開啟老師發放的 W2_Intro_Data.csv，設定三種變項類型：

變項 測量尺度 資料型態 說明

Group Nominal Text 實驗組別（Control / Treatment）

Grade 欄位 尚未存在，需在 Part 4 用計算變項建立

Part 4：計算變項

什麼是計算變項？

根據現有變項，產生新的變項

例如：條件判斷、計算平均值、轉換分數

示範資料：zeppo.csv

操作：☰ → Open → Browse → 選取 zeppo.csv

變項 說明
ID 受試者編號（1–20）
x 測驗分數（50–89）

示範：建立 Pass 計算變項

操作步驟：

  1. 點選 Data → Add → Computed Variable
  2. 輸入新變項名稱：Pass
  3. 在公式欄輸入：
IF(x >= 70, 1, 0)
x 分數 Pass 值 意義
≥ 70 1 及格
< 70 0 不及格

常用計算變項函式語法

IF(條件, 成立時的值, 不成立時的值)
函數 說明
IF() 條件判斷
VMEAN() 變項平均值
VSTDEV() 變項標準差

本週作業（1 小時實作）

作業內容

使用老師發放的 W2_Intro_Data.csv 完成以下任務：

  1. 開啟 W2_Intro_Data.csv（☰ → Open → Browse）
  2. 正確設定三種變項類型：ID → ID、Group → Nominal、Score → Continuous
  3. 新增計算變項 Grade，依 Score 計算等第：
    • ≥ 90 → 1，80–89 → 2，70–79 → 3，< 70 → 4
  4. 確認 Grade 的 Measure Type 設為 Ordinal
  5. 另存為 OMV 格式

作業實作：用 W2_Intro_Data.csv 建立 Grade

請切換至老師發放的 W2_Intro_Data.csv（已在 Part 3 開啟）

根據 Score 計算等第 Grade（用整數表示次序）：

IF(Score >= 90, 1,
  IF(Score >= 80, 2,
    IF(Score >= 70, 3, 4)))
分數範圍 Grade 值 意義
≥ 90 1 最高
80–89 2
70–79 3
< 70 4 最低

建立後：確認 Grade 設為 Ordinal

Grade 有次序（1 < 2 < 3 < 4 代表高到低），但間距不等距

整數計算變項可手動改 Measure Type → Ordinal

操作：雙擊 Grade 欄標題 → Measure Type 改為 Ordinal

評分標準

項目 配分 說明

Grade 為計算變項且設為 Ordinal 20% IF 公式回傳整數 + Measure Type = Ordinal

關鍵概念複習

變項類型判斷口訣

能排序但間距不等 → Ordinal 能加減乘除 → Continuous 只能分類計數 → Nominal 只是標籤編號 → ID

常見錯誤

❌ ID 設為 Continuous（會被當成數字計算）

❌ 月份名稱設為 Continuous（應為 Nominal）

❌ 檔名格式錯誤（無法自動評量）
